# 一. 基础

# 1. 什么是 jsx

JSX是 JavaScript XML, 是React提供的语法糖, 最终运行的时候会经过babel的转义。
使用JSX需要安装相关的语法转换工具, npm i babel-preset-react -D
在根目录的 .babelrc 文件中添加

{
  "presets": ["react"]
}
  • 在JSX语法中： 如果是HTML标签，就按标签解析，如果有{} 则按照变量解析，在{} 中可以写任何符合JS规范的代码
  • 在JSX中，如果要为元素添加Class 属性，因为class在ES6中是关键字，需要用className来代替
  • 在JSX中，如果有for属性，因为label中的for属性是关键字，需要用 htmlFor 代替。
  • 在JSX创建DOM的时候，必须有唯一的元素包裹起来
  • 如果要写注释，需要放到花括号中
{/* 这是注释 */}

# 2. 基本书写的区别

  • class 换成 className ，style 里面放对象
<div className="title" style={{color: 'red', fontSize: 20}}></div>
  • for 要换成 htmlFor，因为for也是关键字
<input type="checkbox" id={"checkbox"}></input>
<label htmlFor={"checkbox"}>这是checkbox</label>
  • 渲染html代码：dangerouslySetInnerHTML
<div dangerouslySetInnerHTML={{ __html: '<p>123</p>' }}></div>
// 渲染结果
<div>
    <p>123</p>
</div>

# 3. 组件的定义

变量的引用只有一个 {}，如：{this.props.name}

  • 函数组件：就是一个函数，返回React元素(虚拟DOM)，首字母需要大写，可以接收参数对象 props(不可更改)。内部没有 state 私有数据，也没有生命周期函数，只有一个 props 来接收外界传递进来的数据。又叫 无状态组件。
// props={number: 0, name: "计算器"}
function Counter(props) {
    return <div>{props.name}：{props.number}</div>
}
// 或者 name='计算器'
ReactDom.render(<Counter number={0} name={'计算器'}>, document.getElementById('root'));
  • 类组件：是一个继承自 React.Component 的一个类，里面必须提供一个 render 方法，需要返回一个 React 元素。组件名称首字母要大写，接收的参数对象 props 不可更改。内部除了有this.props 这个只读属性，还有专门用于存放自己私有数据的 this.state 属性，这个state是可写的。又叫 状态组件。
class Counter extends React.Component {
    render() {
        return <div>{this.props.name}：{this.props.number}</div>
    }
}

# 4. props 的类型检查

(参考珠峰文档)

  • 要在组件的 props 上进行类型检查，只需配置特定的 propTypes 属性
  • 可以通过配置特定的 defaultProps 属性来定义 props 的默认值
import PropTypes from 'prop-types';

class App extends React.Component {
    // 定义 props 的默认值
    static defaultProps = {
        name: "默认值"
    }

    // 配置特定的 propTypes 属性
    static propTypes ={
        name: PropTypes.string.isRequired,
        number: PropTypes.number.isRequired,
        // 这里的函数名可以随便定义，内部会自动走这个函数
        eslint(props, propName, componentName){
            if(props.number<0){
                throw new Error("number must be positive!")
            }
        }
    }
}

# 5. 数据和状态

  • 不要直接修改状态 state，必须要通过 setState
  • 组件在渲染的时候只会修改变动的部分
  • React 的数据是自上而下流动的
  • 想要基于上一个状态来更新的话
this.setState((prevState)=>({number: prevState.number + 1}))
  • 可以通过回调函数来获取最新的state
this.setState({number: this.state.number + 1}, ()=>{
    // 这里的state是最新的状态
})

# 6. ref 的用法

  • 用于获取页面中的 DOM 元素
  • 主流的方式：利用 React.createRef()
class App extends React.Component {
    constructor(props){
        super(props);
        this.num = React.createRef();
    }

    // 其中 input 的 value 值，通过 this.num.current.value 获取
    // 要是想赋值的话，直接 this.num.current.value = newValue
    render(){
        return (
            <div>
                <input ref={this.num}/>
            </div>
        )
    }
}

# 7. 表单受控组件和非受控组件

React受控组件和非受控组件

  • 表单组件分为受控组件和非受控组件
    • 受控组件：使 React 的 state 成为唯一数据源。渲染表单的 React 组件还控制着用户输入过程中表单发生的操作。被 React 以这种方式控制取值的表单输入元素就叫做“受控组件”
    • 非受控组件：表单元素（如input、 textarea 和 select）之类的表单元素通常自己维护 state，并根据用户输入进行更新，这是非受控组件
  • 如果 input 表单只是显示默认值，不会手动去修改该值，则值需要使用 defaultValue，如果需要获取用户输入的值，则一定要用 value 属性，但是一定要配合 onchange 事件一起使用，否则值是不会变的
<input defaultValue={value}/>

# getDerivedStateFromProps 方法

getDerivedStateFromProps 方法

static getDerivedStateFromProps(props, state) 这个生命周期的功能实际上就是将传入的props映射到state上面

WARNING

getDerivedStateFromProps 方法前面必须加 static 关键字，否则会有如下报错

import React from 'react';
import ReactDOM from 'react-dom';
class Counter extends React.Component{ 
    static defaultProps = {
        name:'珠峰架构'
    };
    constructor(props){
        super(props);
        this.state = {number:0}
    }

    handleClick=()=>{
        this.setState({number:this.state.number+1});
    };

    render(){
        console.log('3.render');
        return (
            <div>
                <p>{this.state.number}</p>
                <ChildCounter number={this.state.number}/>
                <button onClick={this.handleClick}>+</button>
            </div>
        )
    }
}
class ChildCounter extends React.Component{
    constructor(props){
        super(props);
        this.state = {number:0};
    }
    static getDerivedStateFromProps(nextProps, prevState) {
        const {number} = nextProps;
        // 当传入的type发生变化的时候，更新state
        if (number%2==0) {
            return { number:number*2};
        }else{
            return { number:number*3};
        }
        // 否则，对于state不进行任何操作
        return null;
    }
    render(){
        console.log('child-render',this.state)
        return (
            <div>
                {this.state.number}
            </div>
        )
    }
}

ReactDOM.render(
    <Counter />,
    document.getElementById('root')
);

# getSnapshotBeforeUpdate 方法

getSnapshotBeforeUpdate 方法

getSnapshotBeforeUpdate() 被调用于render之后，可以读取但无法使用 DOM 的时候。它使您的组件可以在可能更改之前从 DOM 捕获一些信息（例如滚动位置）。此生命周期返回的任何值都将作为参数传递给 componentDidUpdate()

import React from 'react';
import ReactDOM from 'react-dom';
class ScrollingList extends React.Component {
    constructor(props) {
        super(props);
        this.state = {messages: []}
        this.wrapper = React.createRef();
    }

    addMessage() {
        this.setState(state => ({
            messages: [`${state.messages.length}`, ...state.messages],
        }))
    }
    componentDidMount() {
        this.timeID = window.setInterval(() => {//设置定时器
            this.addMessage();
        }, 1000)
    }
    componentWillUnmount() {//清除定时器
        window.clearInterval(this.timeID);
    }
    getSnapshotBeforeUpdate() {//很关键的，我们获取当前rootNode的scrollHeight，传到componentDidUpdate 的参数perScrollHeight
        return this.wrapper.current.scrollHeight;
    }
    componentDidUpdate(perProps, perState, prevScrollHeight) {
        const curScrollTop = this.wrapper.current.scrollTop;//当前向上卷去的高度
        //当前向上卷去的高度加上增加的内容高度
        this.wrapper.current.scrollTop = curScrollTop + (this.wrapper.current.scrollHeight - prevScrollHeight);
    }
    render() {
        let style = {
            height: '100px',
            width: '200px',
            border: '1px solid red',
            overflow: 'auto'
        }
        return (
            <div style={style} ref={this.wrapper} >
                {this.state.messages.map((message, index) => (
                    <div key={index}>{message} </div>
                ))}
            </div>
        );
    }
}

ReactDOM.render(
    <ScrollingList />,
    document.getElementById('root')
);

# 9. Context (上下文)

多个组件需要共享一个状态的话，就需要将状态提升到共同的父组件中，通过上下文 context 来传递数据

  1. 在父组件里面定义 Context ，将定义的 ColorContext 导出
export let ColorContext = React.createContext();  // 返回一个 context 实例
/*
ColorContext 有两个组件
1. Provider 负责提供数据
2. Consumer 负责获取数据
*/
export default class App extends React.Component {
    state = {color: 'red'}
    changeColor = (color) => {
        this.setState({color});
    }
    render(){
        let contextValue = {
            color: this.state.color,
            changeColor: this.changeColor
        }
        return (
            <ColorContext.Provider value={contextValue}>
                <div style={{border: `3px solid ${this.state.color}`, padding: '5px'}}>
                    Panel
                    <Header></Header>
                    <Main></Main>
                </div>
            </ColorContext.Provider>
        )
    }
}
  1. 获取和使用 Context 数据
    • 在子组件中引入 ColorContext
    • 用 StateContext.Consumer 包裹组件
    • 类组件和函数组件都是使用 StateContext.Consumer
import React from 'react';
import { ColorContext } from './App'
import Title from './Title'

export default class Header extends React.Component {
    render(){
        return (
            <ColorContext.Consumer>
            {
                value=>(
                    <div style={{border: `3px solid ${value.color}`, padding: '5px'}}>
                    Header
                    <Title/>
                </div>
                )
            }
            </ColorContext.Consumer>
        )
    }
}
  • 如果父子组件都在同一文件中，获取 context 数据
    • 有两种方式，分别对应类组件和函数组件
    • 其中类组件获取 Context 数据就是给类增加一个静态属性 contextType，变量名 contextType 是固定的，函数组件还是用 ColorContext.Consumer
let ColorContext = React.createContext();
class Header extends React.Component {
    static contextType = ColorContext;
    render(){
        return (
            <div style={{border: `3px solid ${this.context.color}`, padding: '5px'}}>
                Header
                <Title/>
            </div>
        )
    }
}

# 10. 高阶组件

  • 高阶组件一HOC（Highter-Order Components）
  • 高阶组件是为了提高组件的复用率而出现的，抽离出具有相同逻辑或相同展示的组件
  • 高阶组件其实是一个函数，接收一个组件，然后返回一个新的组件，返回的这个新的组件可以对属性进行包装，也可以重写部分生命周期
  • 高阶组件的命名一般都是用with开头
  • 高阶组件基本形式:
const EnhancedComponent = higherOrderComponent(WrappedComponent);
  • 使用例子如下：
import React, { Component } from 'react';
import ReactDOM from 'react-dom';

function withLogger(OldComponent){
    return class extends Component {
        UNSAFE_componentWillMount(){
            // 在组件将要挂载的时候记录一个当前的时间戳
            this.start = Date.now();
        }
        componentDidMount() {
            console.log("本次渲染花了..."+ (Date.now()-this.start)+ "ms")
        }
        render(){
            return <OldComponent {...this.props} />
        }
    }
}

class Counter extends Component {
    componentWillMount() {
        // 在组件将要挂载的时候记录一个当前的时间戳
        this.start = Date.now();
    }
    componentDidMount() {
        console.log("本次渲染花了"+ (Date.now()-this.start)+ "ms")
    }
    render(){
        return (
            <div>
                <p>{this.props.num}</p>
            </div>
        )
    }
}

const CounterWithLogger = withLogger(Counter);
ReactDOM.render(<CounterWithLogger num={100}/>, document.getElementById('root'))

const defaultState = {
    inputValue: '',
    list: [1, 2, 3]
}

export default (state = defaultState, action) => {
    if(action.type === 'change_value'){
        const newState = JSON.parse(JSON.stringify(state));
        newState.inputValue = action.value;
        return newState;
    }
    return state
}
  1. 组件中订阅 store 的变化
constructor(props) {
    super(props);
    // 获取初始 state
    this.state = store.getState();
    // subscribe 方法订阅 store 的变化，调用 handleStoreChange 方法
    store.subscribe(this.handleStoreChange);
}

handleStoreChange = () => {
    this.setState(store.getState());
}
